What are Modules?

  • Modules are non-linked drivers
  • Uses of Modules
    • Allow third party commercial development of drivers
    • Minimise kernel size
    • Allow smaller generic kernels

What is a Module

  • Contains device specific system routines
  • Fills in non-boot gaps in the kernel
  • Can be loaded and removed dynamically

Modules In Linux

  • Modules are compiled per Linux release
  • Multiple modules live in
    • /lib/modules/<kernel version>/
  • Hence only the active version modules will load
  • Accept parameters

Using Modules

  • Building into kernel
    • Make boot only kernel
    • `make modules' to build modules
    • `make module-install' to install modules
  • insmod - Loads a module
  • lsmod - Lists active modules
  • rmmod - removes active modules

Automating Module Use

  • kerneld - kernel module daemon
    • allows run-time loading of modules
  • automatically loads dependent modules
  • will remove modules after timeout
  • can alias devices
    • alias network driver to eth0

Module Applications

  • Allow fun modules
    • installable console screen savers
    • replace console beep
  • Allow conflicting devices to co-exist
