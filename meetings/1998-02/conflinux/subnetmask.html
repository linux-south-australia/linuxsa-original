Subnet Mask

Subnetting - Sub-dividing Networks

The term subnetting refers to taking a single network number (eg. A single Class A network) and sub-dividing it into a number of smaller sub-networks.

Subnetting was introduced to IP addressing to allow better, more efficient allocation of network numbers.

Why Subnet

Before subnetting, each segment in your network (eg. 10Base2 segment) needed an individual network number (ie. its own Class A or B or C number).

If you were using 10Base2, with a limit of 30 ethernet nodes, when you hit this limit, you needed another Class A, B or C network number to use to create another 10Base2 segment. You then connected these two 10Base2 segments together with a device called a router or gateway. In fact, you can only connect 29 Linux boxes to the 10Base2 segment, because the router also counts as an ethernet node.

In the case of a Class A network number, this was a terrible waste of host addresses. A Class A network number can support 16 777 214 host address (3 octets for host part), and you have only used 30, wasting 16 777 184 host addresses. A Class B network is not much better, with a waste of 65 504 host addresses. Even with a Class C, 224 host addresses are wasted.

Subnetting solves this waste problem by using some of the host addresses to represent sub-networks.

How to subnet

To subnet a Class A, B or C network number, you "steal" some of the octets and/or bits available for host addresses, and use them to identify each of your subnets (eg. ethernet segments)

For example, with the Class B address 172.16.0.0 you have 2 octets available for host identification. However, it is a better use of the Class B network if we subnet it by stealing the third octet and using it to identify our subnets.

As a result of this subnetting, we end up with 1 Class B network, with 254 subnets, which can each subnet having 254 hosts attached. The reason why it is 254 subnets and hosts, not 256 subnets and hosts will be explained on the next page.

The subnets will be numbered 172.16.1.0 to 172.16.254.0, with nodes ranging from 1 to 254 in the last octet.

All of the IP devices (including all routers) that are on the same Class A, B or C subnetted network need to be told how the network has been subnetted. A subnet mask is number, with the same format as an IP address, which tells an IP device where the subnet / host boundary is occuring.

Subnet Mask Format

The definition of the subnet mask format is as follows :

  • Same format as an IP address (32 bits, 4 octets, decimal digits, dot separated).

  • Binary 1s for the positions in the IP address corresponding to the network and subnet portions.

  • Binary 0s for the positions in the IP address corresponding to the host portion.

Example Subnet Mask

The earlier example of subnetting Class B network 172.16.0.0 into 254 subnets with 254 hosts per subnet would have a subnet mask of 255.255.255.0 decimal (1111 1111.1111 1111.1111 1111.0000 0000 binary)

Default Subnet Masks

There are two situations when you may not subnet your network number :

  • Your organisation is never going to connect to the Internet (very unlikely!). You then can act like InterNIC, and allocate your own Class A, B and C addresses to your segments.

  • You only have one segment in your organisation (eg. at home). In this situtation though, it may not be very wise to avoid subnetting, because you may need to subnet at a later date, and have to change the subnet masks on all previously configured machines.

In these two cases you use a subnet mask with Binary 1s in the network portion only (as there is no subnet portion!). A subnet mask with no subnet bits is called a default subnet mask.

The default subnet masks for Class A, B and C networks are :

  • Class A - 255.0.0.0

  • Class B - 255.255.0.0

  • Class C - 255.255.255.0
