Getting Linux Connected, using Ethernet and TCP/IP

Common TCP/IP Protocols

There are a large number of TCP/IP protocols implemented at each layer of the TCP/IP model. Some of the most common are :

  • Ethernet / Token Ring / PPP - Technically, these are not really TCP/IP protocols, they are protocols that TCP/IP can operate over.

  • Internet Protocol (IP) - The Internet Protocol is the protocol that is used to deliver the data from the source node to the destination node. It is a best effort, non-reliable (connectionless) protocol. If data gets lost by the Internet Protocol, it is up to the Transport layer protocols to detect this and correct it. Logical identification of devices on the network is specified as part of the Internet Protocol - these addresses are called IP addresses.

  • Transmission Control Protocol (TCP) - The Transport layer protocol provides an end-to-end reliable, flow controlled connection between the end-nodes (it is a connection-oriented protocol). Data sent by the source node should arrive at the end node in the exact same order as it was sent. Because it is connection-oriented, there are three phases of operation - connection establishment, data transfer and connection termination.

  • User Datagram protocol (UDP) - The UDP protocol is an end-to-end protocol, similar to TCP. However, it is a connectionless protocol, meaning that data is sent to the destination node whenever it is ready. This results in better performance, as a connection to the destination does not have to be set up before data transfer. This comes at a cost though, which is a lack of reliability. There is no guarantee that the data will arrive, and arrive in order. The reason it exists is that the programming code to implement UDP is much simpler, and requires less operating memory, allowing it to be embedded in minimal amounts of ROM.

  • Telnet / FTP / TFTP / HTTP - These protocols define how various user applications interoperate. They use the Transport layer protocols to perform this interoperation.

    • Telnet is a virtual terminal protocol, allowing a client node to act as though it was a dumb terminal physically connected to the server node. Telnet uses TCP.
    • FTP is the File Transport Protocol, used to transfer files between a client and server node. FTP uses TCP.
    • TFTP is the Trivial File Transport Protocol, used for transfering boot files to devices booting off of the network. Usually these devices only have a small bootstrap ROM, and therefore TFTP uses UDP.
    • HTTP is the Hyper Text Transport Protocol, used to transfer WWW html files between the WWW client and server. HTTP uses TCP.
